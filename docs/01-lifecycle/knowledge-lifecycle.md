# Knowledge Lifecycle

## Flow

1. Capture raw narrative.
2. Extract structured product knowledge.
3. Classify it as Fact, Decision, Recommendation, Assumption, or Open Question.
4. Store it in the canonical document.
5. Link it to feature specs, issues, code, tests, and release notes when those exist.
6. Check for drift before declaring work complete.

## Canonical Sources

- Product concept: `docs/00-foundation/project-brief.md`
- Feature behavior: `docs/01-lifecycle/features/feature.bookmark-core.md`
- Terms: `docs/00-foundation/glossary.md`
