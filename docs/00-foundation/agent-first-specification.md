# Agent-First Specification Policy

Every meaningful feature should have an Agent-First Specification before implementation.

## Required Fields

- id
- title
- status
- owner
- business_priority
- ai_priority
- automation_level
- human_approval
- audit_required
- source_of_truth
- context
- goals
- non_goals
- personas
- workflow
- acceptance_criteria
- data_contract
- permissions
- configuration
- tests
- docs_impact
- release_impact
- decisions
- assumptions
- open_questions

## Current Feature Specs

- `docs/01-lifecycle/features/feature.bookmark-core.md`

## Rule

If product behavior changes, the feature spec changes first. Code follows accepted knowledge.
